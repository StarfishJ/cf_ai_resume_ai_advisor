## 8. Ownership & Action Verbs
*HM/HR Tip: Same principles apply — show ownership, not passivity.*
- **High Impact (Use these)**: Designed, Implemented, Optimized, Developed, Accelerated, Architected, Profiled, Debugged, Instrumented, Built, Led, Owned
- **Low Impact (Avoid these)**: Participated in, Assisted with, Helped, Learned, Exposed to, Responsible for (passive)

## 9. Quantifiable Metrics (Results-Oriented)
*HM/HR Tip: Amazon is data-driven. ML systems roles especially value performance numbers.*
- **Performance**: Training throughput (samples/sec), Inference latency (ms), Speedup (X faster), FLOPS utilization (%)
- **Efficiency**: Memory usage optimization (%), Power optimization (W), Cost reduction ($/%)
- **Scale**: Model size (B parameters), Cluster size (N nodes/GPUs), Dataset size (TB)
- **Quality**: Accuracy improvement (%), Test coverage (%), Bug reduction
- **Example Formats**: "Achieved Xx speedup in distributed training", "Reduced inference latency by X%", "Optimized memory usage by X% for B-parameter model"

## 10. Amazon Leadership Principles Signals
*HR/HM Tip: Same LP signals as backend, equally important for ML systems roles.*
- **Customer Obsession**: AWS customer impact, user-facing ML services, real-world deployment
- **Ownership**: End-to-end system design, full-stack ownership (hardware to software)
- **Bias for Action**: Rapid prototyping, shipped, iterated
- **Dive Deep**: Root cause analysis, performance profiling, deep hardware-software debugging
- **Invent and Simplify**: Novel architecture, simplified pipeline, automated workflow
- **Deliver Results**: Measurable speedup, shipped to production, met performance targets
